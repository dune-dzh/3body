Three-body problem — computational solution

The trajectory you see is a numerical solution: Newtonian gravity in the plane, stepped in time on the server and streamed here for visualization — not a closed-form formula (those are rare for n = 3).

What “the solution” is on this page

Solution here means the time-evolving positions of three masses that satisfy Newton’s laws, computed approximately: we cannot write one neat function r(t) for arbitrary three-body setups, so the program integrates the equations of motion step by step. The canvas is that approximate trajectory over time.

Schematic: every pair attracts. The actual run uses the same pairwise idea in 2D with masses and speeds you can change only in code — the UI explains the method, not a parameter panel.

Model & integration (matches the server)

State for each body i: position (x, y) and velocity (vx, vy). Constants in code use G = 1 and a small softening ε so accelerations stay finite when two points get very close.

Acceleration of body i from body j (i ≠ j):

ai←j = G · mj · (rj − ri) / (‖rj − ri‖² + ε)3/2

Sum the contributions from both other bodies, then each step:

v ← v + a·Δt
x ← x + v·Δt

Several small Δt steps are taken between frames you receive over the WebSocket, so the orbit shape is a faithful sample of that discrete solution, not an artistic sketch.

Guided tour — step through how this connects to the picture on the right.

Each color is one mass; bright disks are its current place in the numerical solution. Trails are recent past positions — a visible history of the integrated trajectory. Chaos: for three bodies, tiny changes in start state can produce wildly different paths (same equations, different outcome).
